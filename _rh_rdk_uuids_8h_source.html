201 /** Note that if you want to support the Linear Workflow settings in Rhino, you will need to do the following in your renderer:
202 
203  1) Return \e true for uuidFeatureLinearWorkflow in your CRhRdkPlugIn::SupportsFeature override.
204 
205  2) Check that Linear Workflow is on using IRhRdkLinearWorkflow::On(), and if so:
206 
207  3) Convert all colors that you get from Rhino to their linear equivalent using
208  CRhRdkColor::ApplyLinearWorkflowGamma() / IRhRdkLinearWorkflow::ApplyLinearWorkflowGamma()
209 
210  4) Convert all texture colors *excluding HDR textures* (those that return true for CRhRdkTexture::IsLinear), or textures
211  that are used in a bump or normal map using CRhRdkColor::ApplyLinearWorkflowGamma().
212  \note You can call CRhRdkMaterial::TextureRequiresLinearWorkflowAdjustment() to determine if the texture requires it.
213 
214  5) If you are not using the built-in render window, apply the gamma value in IRhRdkLinearWorkflow::Gamma().
215  \note While Rhino 5 automatically did some of the gamma correction work in SimulateMaterial(), this is not the case
216  in Rhino 6. Everything now has to be done explicitly by the client.
217 */
