6 /** \class CRhRdkContentFactory

8  This class is a factory for producing a certain type of content.
9  You must create and implement a subclass for each content type that your renderer supports.
10 
11  Content factories are RDK extensions. You usually add new content factories to the system when you
12  are called on your implementation of CRhRdkPlugIn::RegisterExtensions(). If necessary, you can also
13  add new factories later by calling RhRdkAddExtension(). Factories so added can be found again later
14  by calling RhRdkContentFactories() which returns an IRhRdkContentFactories interface.
15 
16  When you add a factory to the factory collection, a set of tests are performed on the factory
17  before it is allowed to be added. The factory is only added if the following succeeds:
18 
19  - CRhRdkContentFactory::Initialize() must return \e true.
20  - CRhRdkContentFactory::NewContent() must return a valid content.
21  - CRhRdkContent::Initialize() must return \e true.
22 
23  If these tests succeed, RDK takes ownership of the factory object and sends the
24  OnFactoryAdded event. Otherwise, the factory is considered bad and is deleted.
