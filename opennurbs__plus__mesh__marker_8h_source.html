104  /*
105  Description:
106  Transfer initial marks to connected components.
107  Parameters:
108  bUseTopologicalComponent - [in]
109  If true, then vertices that are at the same location
110  are treated as one vertex. Setting this value to
111  true requires more time and memory to be used. The
112  additional time expense is quick sort of m_V_count
113  items. The additional memory expense is two integer
114  heap arrays of length m_V_count.
115  If false, then the vertex point information is ignored
116  and each vertex is considered to be distinct.
117  bProcessUnmarkedVertices - [in]
118  A mesh may have isolated vertices (not used by a face) and
119  some of these may not be marked by calls to MarkVertex().
120  When bProcessUnmarkedVertices=true, if at the end of the
121  component marking calculation there are unmarked vertices,
122  then a new mark value is generated and assigned to each
123  unmarked vertex. If bProcessUnmarkedVertices=false, then
124  vertices that are not marked by the calculation are left
125  unmarked.
126  bProcessUnmarkedFaces - [in]
127  Generally a mesh has several components and some may have
128  no marks in them. When bProcessUnmarkedFaces=true, a
129  new mark value is generated for one face in each unmarked
130  component and then the entire component the face belongs to
131  is marked with that value. When bProcessUnmarkedFaces=false,
132  if at the end of the marking calculation there are unmarked
133  components, then those components are left unmarked.
134  If you set bProcessUnmarkedFaces=false, then you must use
135  MarkFace() or MarkVertex() to set a see mark in at least
136  one component before calling MarkComponents().
137  Returns:
138  True if mesh is successfully marked. The mark informtion is
139  is returned in the members
140 
141  m_vertex_mark_value[]
142  m_face_mark_value[]
143  m_marked_vertex_count
144  m_marked_face_count
145  m_deleted_vertex_count
146  m_deleted_face_count
147  m_max_mark_value
148 
149  in the m_face_mark_value[] and m_vertex_mark_value[] arrays.
150  These arrays have length m_F_count and m_V_count, respectively,
151  and are parallel to the m_F[] and m_V[] arrays.
152  m_marked_vertex_count is the number of non-zero (positive or
153  negative) elements m_vertex_mark_value[].
154  m_marked_face_count is the number of non-zero (positive or
155  negative) elements m_face_mark_value[].
156  m_deleted_face_count is the number of negative
157  elements in m_face_mark_value[].
158  m_deleted_vertex_count is the number of negative
159  elements in m_vertex_mark_value[].
160  The maximum mark value used is in m_max_mark_value.
161 
162  MarkComponents() increments m_marking_error_count everytime it
163  encounters an error condition that it can tolerate.
164  If MarkComponents() returns true and m_marking_error_count is
165  greater than 0, then exercise caution using the results.
166  */
