672 /*
673 Description:
674  Mesh Rhino objects.
675 Parameters:
676  objects - [in] objects to get render meshes for.
677  render_meshes - [out]
678  render_meshes[i].Object() = input object
679  render_meshes[i].Mesh() = render mesh
680  bOkToCreate - [in] if true and the render meshes do not
681  exist, then they will be created.
682  bReturnAllObjects - [in] curves, points, etc., do not
683  have render meshes. If you want these objects to
684  appear in the output list with a NULL mesh, then
685  set this flag to true.
686  bSkipHiddenObjects - [in] will not create render meshes for
687  hidden objects if set to true.
688 Returns:
689  Number of meshes appended to render_meshes[].
690 Remarks:
691  References to instance definitions that contain multiple
692  objects will appear multiple times in the output list.
693 
694  When the input object is a CRhinoInstanceObject, one or
695  more CRhinoObjRefs may be returned in the render_meshes[]
696  array. Each atomic object the instance reference ultimately
697  refers to gets a CRhinoObjRef. The Object() returned by
698  the CRhinoObjRef points to the atomic object in the
699  instance definition table. The Geometry() returned by the
700  CRhinoObjRef is a transformed copy of the mesh. The mesh
701  is correctly located and does not require further
702  transformations. In this case,
703  CRhinoObjRef::GetInstanceTransformation() will
704  return a pointer to the top level instance reference
705  (the input CRhinoInstanceObject) and report the
706  total transformation that was applied to the mesh.
707  In the case of nested instance references, this
708  transformation may be different from the transformation
709  associated with the input CRhinoInstanceObject.
710 */
711 RHINO_SDK_FUNCTION
712 int RhinoGetRenderMeshes(
713  const ON_SimpleArray< const CRhinoObject* >& objects,
714  ON_ClassArray< CRhinoObjRef >& render_meshes,
715  bool bOkToCreate,
716  bool bReturnAllObjects,
717  bool bSkipHiddenObjects
