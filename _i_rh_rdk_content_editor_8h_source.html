47 /** \class IRhRdkContentEditor
48 
49  This interface represents a content editor. Currently two types of content editor are implemented:
50 
51  1. The Thumbnail Editor, of which there are 3 kinds, Material Editor, Environment Editor
52  and Texture Palette.
53 
54  2. The In-place Editor which is found in the Object Properties material page,
55  the Layer Material dialog and the Ground Plane dialog.
56 
57  A content editor acts as a host for a content UI. The UiHostWindow() method
58  returns a pointer to the window that is the parent of the main UI window.
59  The main UI window is usually a content UI holder containing sections
60  (AKA roll-ups), but if you override CRhRdkCoreContent::CreateUI() and provide
61  a custom UI, it can be anything you like.
62 
63  A content editor must know the document it is for. This is not strictly needed on Windows,
64  but on the Mac there can be many editors for a content kind but each for a different
65  document. Even modal editors which are capable of editing free-floating contents
66  still have a document, but this document is a 'sandbox', not a 'live' document.
67 
68 */
