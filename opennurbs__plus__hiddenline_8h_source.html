55  // Step II. Specify viewport
56  /*
57  Description:
58  Set the viewport for the hidden line drawing. The viewport supplies the projection that determines
59  the visibility of curves, and the HiddenLineDrawing coordinate system in which the resulting
60  ON_HLDCurve's are represented. The HLD-coordinate system is a right handed system
61  for 3-d model space, in which the 3rd coordinate direction is the projection
62  direction. In particular the z-coordinate direction points from the scene toward the camera.
63  Parameters:
64  v -[in] viewport. A copy of the viewport is made inside of ON_HiddenLineDrawing.
65  Returns:
66  true if viewport has been set
67  Details:
68  Only allowed before calling Draw(..).
69 
70  For perspective views the "target depth" is used to determine the scale of the results.
71  The target_depth, v.TargetDistance(false), defines the target plane that is the plane
72  perpendicular to the camera axis at this distance from the camera location.
73  The HLD-Coordinate system is scaled 1:1 to world coordinates on the target plane.
74  The HLD_coordinate system is translated in the z-direction so that the target plane
75  has z=0 in HLD-coordinates.
76  If you have a viewport and don't want to think about the choice of target_depth
77  you might want to try:
78  double near = v.FrustumNear();
79  double far = v.FrustumFar();
80  double target_distance = .5*(near + far);
81  bool rc = v.SetFrustumNearFar( near, far, 0.0, 0.0, target_distance);
82  if( rc)
83  HLD.SetViewport(v);
84 
85  Another option is to leave the target_depth unset, or more explicity call
86  v.SetTargetPoint( ON_UNSET_POINT)
87  When the taget point is unset, a target_depth will be computed from the geometry and used to define
88  the HLD coordintate system.
89 
90  Perspective distortion ( on perspective views only) will scale objects in front (back) of the target plane
91  to be larger (smaller) in HLD-coordinates than in world coordinates. The absolute tolerance is applied in
92  HLD-coordinates so its effect depends on the target_depth setting.
93 
94  If v is a parallel projection viewport the HLD-coordinate system is the camera coordinate system of v.
95 
96  For perspective views the transformation from world coordinates to HLD-coordinates is
97  a diagonal scaling of the view transformation from world to clip coordinates.
98  Specifically
99  [ W/2 ]
100  World_to_HLD-coordinates = [ H/2 ] x World_to_clip_coordinates
101  [ (f - n)/2 d ]
102  [ 1.0 ]
103 
104  where W and H are the width and depth of the view frustum at the target_depth and f and n are the
105  distances to the far and near clipping plane in the viewport, d is chosen so that the target plane
106  is represented as the plane z=0 in HLD-coordinates.
107  */
108  bool SetViewport(const ON_Viewport& v);
