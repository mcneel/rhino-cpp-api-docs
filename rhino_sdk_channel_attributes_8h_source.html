16 // Flags and attributes need to be provided to pass back and forth to the
17 // conduits based on all of the default stuff the pipeline draws in
18 // its channels.
19 //
20 // Reason: Conduits may want to override the default behavior (eg. DrawGrid),
21 // so we need a way for conduits to tell the pipeline what is being
22 // done, as well as let other conduits know what has already been
23 // done.
24 //
25 // Note: These are all based on what Rhino's pipeline does...there is no
26 // way to second guess what other pipelines will or won't do, so all
27 // we can do for now is tell conduits as much as we can about Rhino's
28 // pipeline attributes...
