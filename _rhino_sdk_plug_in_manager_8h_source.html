600  // Added by Dale Lear 25 February 2004
601  //
602  // In SDK's with RHINO_SDK_SERVICE_RELEASE >= 200402250.
603  // We need a way for Rhino to prevent loading of certain plug-ins
604  // that are known to cause trouble. The default load protection
605  // is 0. If a user decides a certain plug-in is causing trouble,
606  // they can use the plug-in manager to add load protection.
607  // The reason that both the 0 and 1 values mean silent loading
608  // is so that we don't waste time saving the default information
609  // in registry unless we are actually dealing with a plug-in
610  // where the user has explicitly changed the load protection.
611  //
612  // 0 = silently load - (default setting)
613  // 1 = silently load - (setting came from registry)
614  // 2 = query load (yes/no modal message box)
615  //
616  // The PlugInManager command provides UI for changing setting
617  // the load protection to silent load or query load.
618  //
619  // The non-default load permissions are stored in
620  // HKEY_CURRENT_USER\...\Global Settings\Plug-ins<plug-in uuid>\LoadProtection
621  // so that ordinary users (non-administrators) can control loading of
622  // buggy plug-ins.
623  //
624  // Use CRhinoPlugIn::SaveLoadProtectionToRegistry(plug-in_uuid,m_LoadProtection)
625  //
626  // to save this value if it is ever changed.
627  //
628  // ALERT - snotty comment follows:
629  // Because the CRhPlugInManager has become such a twisted pile of crap,
630  // this value has to be stored on 2 different classes:
631  // CRhinoPlugInRecord::m_nLoadProtection
