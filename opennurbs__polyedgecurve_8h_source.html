330  // When m_component_index refers to an ON_BrepTrim, there
331  // are four domains and 4 classes derived from ON_Curve
332  // that play a role in the polyedge segment. It is possible
333  // for all 4 of these domains to be different.
334  //
335  // "this" ON_PolyEdgeSegment is an ON_ProxyCurve. The
336  // evaluation domain of "this" is
337  // = this->Domain()
338  // = ON_ProxyCurve::m_this_domain
339  //
340  // ON_ProxyCurve::m_real_curve points to the curve in the
341  // c3 = ON_Brep::m_C3[edge.m_c3i]. "this" is a proxy for some
342  // sub-interval of c3.
343  // = this->ProxyCurveDomain()
344  // = ON_ProxyCurve::m_real_curve_domain
345  //
346  // The edge, an ON_BrepEdge, is also a proxy based on c3,
347  // and the edge's evaluation domain is edge.m_this_domain.
348  // ON_PolyEdgeSegment::m_edge_domain records the increasing
349  // subinterval of edge.m_this_domain that corresponds
350  // to the portion of c3 "this" is using.
351  //
352  // The trim, an ON_BrepTrim, is a proxy based on a curve
353  // in ON_Brep::m_C2[]. Some portion of the trim corresponds
354  // to the portion of the edge we are using. m_trim_domain
355  // is an increasing, possible subinterval, of the trim's domain
356  // ON_BrepTrim::m_this_domain.
