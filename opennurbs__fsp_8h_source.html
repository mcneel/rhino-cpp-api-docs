30  /*
31  Description:
32  Create a fixed size memory pool.
33  Parameters:
34  sizeof_element - [in]
35  number of bytes in each element. This parameter must be greater than zero.
36  In general, use sizeof(element type). If you pass a "raw" number as
37  sizeof_element, then be certain that it is the right size to insure the
38  fields in your elements will be properly aligned.
39  element_count_estimate - [in] (0 = good default)
40  If you know how many elements you will need, pass that number here.
41  It is better to slightly overestimate than to slightly underestimate.
42  If you do not have a good estimate, then use zero.
43  block_element_capacity - [in] (0 = good default)
44  If block_element_capacity is zero, Create() will calculate a block
45  size that is efficent for most applications. If you are an expert
46  user and want to specify the number of elements per block,
47  then pass the number of elements per block here. When
48  block_element_capacity > 0 and element_count_estimate > 0, the first
49  block will have a capacity of at least element_count_estimate; in this
50  case do not ask for extraordinarly large amounts of contiguous heap.
51  Remarks:
52  You must call Create() on an unused ON_FixedSizePool or call Destroy()
53  before calling create.
54  Returns:
55  True if successful and the pool can be used.
56  */
57  bool Create(
58  size_t sizeof_element,
59  size_t element_count_estimate,
60  size_t block_element_capacity
61  );

85  /*
86  Description:
87  Return an element to the pool.
88  Parameters:
89  p - [in]
90  A pointer returned by AllocateElement().
91  It is critical that p be from this pool and that
92  you return a pointer no more than one time.
93  Remarks:
94  If multiple threads are using this pool, then use ThreadSafeReturnElement().
95 
96  If you find the following remarks confusing, but you really want to use
97  ReturnElement(), then here are some simple guidelines.
98  1) SizeofElement() must be >= 16
99  2) SizeofElement() must be a multiple of 8.
100  3) Do not use FirstElement() and NextElement() to iterate through
101  the pool.
102 
103  If 1 to 3 don't work for you, then you need to understand the following
104  information before using ReturnElement().
105 
106  ON_FixedMemoryPool uses the first sizeof(void*) bytes of the
107  returned element for bookkeeping purposes. Therefore, if you
108  are going to use ReturnElement(), then SizeofElement() must be
109  at least sizeof(void*). If you are using a platform that requires
110  pointers to be aligned on sizeof(void*) boundaries, then
111  SizeofElement() must be a multiple of sizeof(void*).
112  If you are going to use ReturnElement() and then use FirstElement()
113  and NextElement() to iterate through the list of elements, then you
114  need to set a value in the returned element to indicate that it
115  needs to be skipped during the iteration. This value cannot be
116  located in the fist sizeof(void*) bytes of the element. If the
117  element is a class with a vtable, you cannot call a virtual
118  function on a returned element because the vtable pointer is
119  trashed when ReturnElement() modifies the fist sizeof(void*) bytes.
120  */
121  void ReturnElement(void* p);
122

391  Sets the state of the iterator to the initial state that

401  /*
402  Description:
403  Get a pointer to the first element in the first block.
404  Parameters:
405  block_element_count - [out] (can be null)

407  first block is returned in block_element_count.

411  The loop will iteratate through all the blocks.

452  See the FirstBlock() documentation.

454  The next block when iterating through the blocks.

459  void* NextBlock( size_t* block_element_count );
