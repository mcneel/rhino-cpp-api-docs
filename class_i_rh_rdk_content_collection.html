Detailed Description

This interface is returned by GetData(uuidData_RdkContentDatabase) where it will get a collection that represents all the content in a 'database' – usually a document.

It is also returned by GetData(uuidData_RdkContentDisplayCollection) where it will get a collection that represents only the content in a database that is not filtered out by a search phrase.

It is also returned by GetData(uuidData_RdkContentSelection) where it will get a collection that represents the current selection in a UI.

Get a new content iterator. The caller shall delete the iterator after use.

Note
It is the caller's responsibility to filter out unwanted contents from the returned collection. To filter out the default instance, you can call CRhRdkContent::IsDefaultInstance() and ignore the content if the result is true. To filter out proxies, you can call CRhRdkContent::ProxyType() and ignore the content if the type is not CRhRdkContent::ProxyTypes::None.

◆ SetSearchPattern()

virtual void IRhRdkContentCollection::SetSearchPattern ( const wchar_t *  wszPattern)
pure virtual

Sets a search pattern for filtering contents. This is not actually used by the iterator, but is stored for use by any UI that wants to filter contents based on a search string by using the function RhRdkCheckSearchPattern().

Parameters
wszPatterncan be a plain string or one of the following: name: <name> note: <note> (or notes: <note>) tags: <tags> (or tag: <tags>) This search pattern is used by RhRdkCheckSearchPattern() in the following way: If pattern is a plain string, the name, notes and tags are searched for any occurrence of the string. If the pattern starts with name:, a wildcard search is performed on the name parameter only. If the pattern starts with note:, a wildcard search is performed on the notes parameter only. If the pattern starts with tags:, a wildcard search is performed on each semicolon-delimited tag in the tags parameter only.
