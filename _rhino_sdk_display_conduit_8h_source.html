13 // Specialiazed UINT class that ensures "supported" values only...
14 //
15 // The display pipeline calls conduits during specific phases of drawing (channels)
16 // During the drawing of a single frame the channels are called in the following order
17 //
18 // [Begin Drawing of a Frame]
19 // SC_CALCBOUNDINGBOX
20 // SC_CALCCLIPPINGPLANES
21 // SC_SETUPFRUSTUM
22 // SC_SETUPLIGHTING
23 // SC_INITFRAMEBUFFER
24 // SC_DRAWBACKGROUND
25 // If this is a layout and detail objects exist the channels are called in the
26 // same order for each detail object (drawn as a nested viewport)
27 // SC_PREDRAWMIDDLEGROUND
28 // SC_PREDRAWOBJECTS
29 //
30 // For Each Visible Non Highlighted Object
31 // SC_OBJECTDISPLAYATTRS
32 // SC_PREOBJECTDRAW
33 // SC_DRAWOBJECT
34 // SC_POSTOBJECTDRAW
35 // Next Object
36 //
37 // Here Rhino draws highlighted objects
38 //
39 // SC_PREDRAWTRANSPARENTOBJECTS
40 //
41 // Here Rhino draws transparent objects (sorted)
42 //
43 // SC_POSTDRAWOBJECTS
44 // SC_DRAWFOREGROUND
45 //
46 // For Each Visible Highlighted Object
47 // SC_OBJECTDISPLAYATTRS
48 // SC_PREOBJECTDRAW
49 // SC_DRAWOBJECT
50 // SC_POSTOBJECTDRAW
51 // Next Object
52 //
53 // SC_POSTPROCESSFRAMEBUFFER - If a conduit exists that requires this
54 // SC_DRAWOVERLAY - If Rhino is in a feedback mode
55 // [End of Drawing of a Frame]
56 //
57 // NOTE: There may be multiple SC_DRAWOBJECT calls for a single object. An example of when this could
58 // happen would be with a shaded sphere. The shaded mesh is first drawn and these channels would
59 // be processed; then at a later time the isocurves for the sphere would be drawn.

562 //////////////////////////////////////////////////////////////////////////
563 //////////////////////////////////////////////////////////////////////////
564 // Specialized abstract conduit class that provides very high level support
565 // for setting up and managing 3D Stereo viewports.
566 //
567 // There are basically two types of stereo modes supported in Rhino:
568 // 1) Hardware shutter glasses
569 // 2) Some kind of software generated stereo view,
570 // (i.e. Anaglyph red/blue glasses)
571 //
572 // You cannot support both types in one conduit. When you construct your
573 // stereo conduit you must specify what type you plan on supporting and
574 // all the appropriate setup and framework initialization will be done
575 // for you automatically.
576 //
577 // Each "stereo phase" is broken into 3 primary parts (or channels):
578 // 1) Frame buffer initialization
579 // 2) Frustum setup and initialization
580 // 3) Frame buffer drawing
581 //
582 // ...each of those phases is called for each eye and are handled in
583 // the abstract routines:
584 //
585 // ProcessLeftEye(...)
586 // ProcessRightEye(...)
587 //
588 // For obvious reasons, you should process all left eye calculations and
589 // left eye drawing inside "ProcessLeftEye", and all right eye calculations
