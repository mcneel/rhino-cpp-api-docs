27 /*
28 Descripton:
29  ON_RegKey provides simplified access to the Windows Registry.
30  All functions return true to indicate success, false for failure.
31  If a function returns false, check the ErrorCode property to see extended
32  error codes returned by the underlying Windows API functions.
33 
34  Performance Note: The underlying Windows HKEY is not cached; it is opened
35  and closed for each operation.
36  No testing has been done to determine if this causes much overhead for
37  lots of reads and writes.
38  The reason for this is to simplify the underlying implementation as copy
39  constructors and operator = do not need to
40  check for open keys, close them when necessary, or worry about other
41  copies closing the handle.
42 */

82  /*
83  Description:
84  OpenRead opens the specified registry key for reading. It optionally creates the key if it does not exist, then opens it for reading.
85  Parameters:
86  KeyName - [in] the full path to the registry key, including the hive. The hive can be the full name or
87  abbreviation: HKEY_LOCAL_MACHINE or HKLM, HKEY_CURRENT_USER or HKCU, HKEY_CLASSES_ROOT or HKCR,
88  HKEY_CURRENT_CONFIG or HKCC, HKEY_USERS or HKU.
89  See examples for details.
90  Create - [in, optional] if true, KeyName will be created if it doesn't already exist. Default is false.
91  Returns:
92  true when SubKey could be created or opened.
93  false otherwise
94  ErrorCode contains the return value of the underlying call to RegOpenKeyEx or RegCreateKeyEx
95  Examples:
96  // Open HKEY_LOCAL_MACHINE\\Software\\MyCompany\\MyProduct if it exists
97  ON_RegKey reg;
98  reg.Open(L"HKEY_LOCAL_MACHINE\\Software\\MyCompany\\MyProduct");
99 
100  // Create HKEY_LOCAL_MACHINE\\Software\\MyCompany\\MyProduct
101  ON_RegKey reg(L"HKEY_LOCAL_MACHINE\\Software\\MyCompany\\MyProduct", true);

103  // Open HKEY_LOCAL_MACHINE\\Software\\MyCompany\\MyProduct
