47  // This version of fit exposes all of the controls
48  // that the rhino patch command uses.
49  // input_geometry: Combination of ON_Curves, ON_BrepTrims, ON_Points, ON_PointClouds or ON_Meshes
50  // pSrf: Starting surface or NULL to use a plane fit through the input points
51  // u_spans,
52  // v_spans: Number of surface spans used when a plane is fit through points to start
53  // bTrim: If true, try to find an outer loop from among the input curves and trim the result to that loop
54  // bTangency: If true, try to find brep trims in the outer loop of curves and try to fit to the
55  // normal direction of the trim's surface at those locations.
56  // point_spacing: Basic distance between points sampled from input curves.
57  // flexibility: Determines the behavior of the surface in areas where its not otherwise controlled
58  // by the input. Lower numbers make the surface behave more like a stiff material,
59  // higher, more like a stiff material
60  // That is, each span is made to more closely match the spans adjacent to it if there
61  // is no input geometry mapping to that area of the surface when the flexibility value
62  // is low. The scale is logrithmic. Numbers around 0.001 or 0.1 make the patch pretty stiff
63  // and numbers around 10 or 100 make the surface flexible.
64  // surface_pull: Tends to keep the result surface where it was before the fit in areas where
65  // there is on influence from the input geometry
66  // fix_edges[]: Flags to keep the edges of a starting (untrimmed) surface in place while fitting
67  // the interior of the surface. Order of flags is left, bottom, right, top
68  // tolerance: Tolerance used by input analysis functions for loop finding, trimming, etc.
69  static
70  ON_Brep* Fit(
71  ON_SimpleArray<const ON_Geometry*>& input_geometry,
