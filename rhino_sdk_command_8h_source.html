33  /*
34  m_style determines what kind of user interface (ui) the command
35  can use and how serious errors should be reported.
36 
37  interative_ui:
38  Can use dialogs for input. Must use message boxes to report
39  serious error conditions.
40 
41  script_ui:
42  All input must come from command line, GetPoint, GetObject,
43  GetString, etc. Must use message boxes to report serious
44  error conditions. Script mode gets used when a command is
45  run with a hyphen (-) prefix.
46 
47  batch_ui:
48  All input must come from command line, GetPoint, GetObject,
49  GetString, etc. Must use RhinoApp().Print() to report serious
50  error conditions. NO dialog boxes are permitted.
51  Batch mode is enabled by calling the Rhino BatchModeOn
52  command in a command script.
53 
54  */

181  /*
182  Description:
183  Rhino calls uses the virtual ReplayHistory functions to
184  to remake an objects when inputs have changed.
185  If a command supports history, it must override exactly
186  one of the ReplayHistory functions. Simple commands
187  can use the simple version of replay history that just
188  just generates a new object. Complex commands, like
189  Project and Interesect need to override the complex
190  version of replay history that uses a results[] array
191  to return multiple objects.
192  Parameters:
193  history - [in]
194  History information the command record when it first
195  created the existing objects.
196 
197  results - [in/out] (complex version)
198  results[i].m_objects[0] - [in] existing object
199  results[i].m_objects[1] - [out] updated object
200 
201  The input array has the objects that were
202  created by the command in results[i].m_objects[0].
203 
204  ReplayHistory is responsible for creating replacements
205  and putting them in results[i].m_objects[1]. The replacements
206  must NOT be added to the document.
207  The results[] array can be grown if there are new "updated" objects
208  that do not replace an existing object. In this case
209  set the m_objects[0] pointer to NULL. An input
210  results[i].m_objects[0] can be set to NULL if no replacement
211  can be calculated.
212 
213  If results[i].m_objects[0] and results[i].m_objects[1] are
214  both not NULL, then m_objects[1] will replace m_objects[0].
215  If results[i].m_objects[0] is not NULL and results[i].m_objects[1]
216  is NULL, then results[i].m_objects[0] is deleted.
217  If results[i].m_objects[0] is NULL and results[i].m_objects[1]
218  is not NULL, then results[i].m_objects[1] is added.
219  Returns:
220  The simple version returns either NULL or an updated object.
221  The complex version returns true if Rhino should process
222  the information in results[] and false otherwise.
223  */
224  virtual
225  bool ReplayHistory(
226  const class CRhinoHistoryRecord& history_record,
227  ON_SimpleArray<CRhinoObjectPair>& results
228  );
229 
230  virtual
231  CRhinoObject* ReplayHistory(
232  const class CRhinoHistoryRecord& history_record

242  history - [in]

245

247  results[i].m_objects[0] - [in] existing object
248  results[i].m_objects[1] - [out] updated object
249 
250  The input array has the objects that were
251  created by the command in results[i].m_objects[0].
252

254  and putting them in results[i].m_objects[1]. The
255  replacements must NOT be added to the document.
256  An input results[i].m_objects[0] can be set to NULL

260  both not NULL, then m_objects[1] will replace m_objects[0].
261  If results[i].m_objects[1] is NULL, then nothing is done

263  If results[i].m_objects[0] is NULL and results[i].m_objects[1]
