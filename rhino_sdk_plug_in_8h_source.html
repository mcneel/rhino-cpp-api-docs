20 // class CRhinoPlugIn
21 //
22 // Description:
23 // This is the base class for every Rhino plug-in type. Never
24 // derive directly from this class use one of the following:
25 // CRhinoUtilityPlugIn
26 // CRhinoFileImportPlugIn
27 // CRhinoFileExportPlugIn
28 // CRhinoDigitizerPlugIn
29 // CRhinoRenderPlugIn
30 // CRhinoDisplayPlugIn
31 //
32 // Remarks:
33 // A plug-in is a DLL and MUST be dynamically linked with MFC
34 // DLLs.
35 //
36 // Because a plug-in DLL is dynamically linked against the MFC
37 // DLLs, any functions exported from this DLL which
38 // call into MFC must have the AFX_MANAGE_STATE macro
39 // added at the very beginning of the function.
40 //
41 // Example:
42 //
43 // extern "C" BOOL PASCAL EXPORT ExportedFunction()
44 // {
45 // AFX_MANAGE_STATE(AfxGetStaticModuleState());
46 // // normal function body here
47 // }
48 //
49 // It is very important that this macro appear in each
50 // function, prior to any calls into MFC. This means that
51 // it must appear as the first statement within the
52 // function, even before any object variable declarations
53 // as their constructors may generate calls into the MFC
54 // DLL.
55 //
56 // Please see MFC Technical Notes 33 and 58 for additional
57 // details.
58 //
