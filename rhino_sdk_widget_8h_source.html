170 /*
171 Description:
172 
173  CRhinoWidget is a special kind of 'phantom' Rhino object which is used for widgets.
174  A widget is a tool the user can use to manipulate data such as texture mappings or decals.
175  Because widgets are also Rhino objects, they work with the gumball.
176 
177  The widget base class is intentionally 'dumb'. A widget doesn't know exactly what it is being
178  used for. That knowledge is delegated to subclasses created by widget clients.
179 
180  NOTE: This object evolved from the 'mapping widget' but it is really nothing to do
181  with 'mapping', 'texture mapping' or ON_TextureMapping. Mapping-specific code must not
182  appear in this base class or its implementation.
183 
184  To create your own widget, derive a class from CRhinoWidget and create the widget as follows:
185 
186  ::RhCreateWidget(doc, new CMyWidget(...));
187 
188  Rhino will create a document-resident widget from your instance and take ownership of it; you
189  must not store a pointer to it. Widgets, like other document-resident Rhino objects, are managed
190  by the document.
191 
192 */
