378  // This field is used for textures with type
379  // bitmap_texture that reference bitmap files that do
380  // not have an alpha channel and is used to set
381  // runtime alpha values. It needs to be parsed when the
382  // texture is loaded and can be ignored at runtime.
383  //
384  // If ON_UNSET_COLOR != m_transparent_color, then
385  // a pixel in the bitmap file with a matching RGB
386  // value is assigned the alpha value (ON_Color::Alpha)
387  // in m_transparent_color. The intended use is
388  // for non-rectangular decals defined by RGB bitmaps in
389  // files that don't save an alpha channel.
390  //
391  // For example if the decal is a red number 7 with a
392  // white background, then you would set m_transparent_color's
393  // RGB to white and its A to zero.
394  ON_Color m_transparent_color = ON_UNSET_COLOR;
395 
396  // This field is used for textures with type
397  // bitmap_texture that reference bitmap files that do
398  // not have an alpha channel and is used to set
399  // runtime alpha values. It needs to be parsed when the
400  // texture is loaded and can be ignored at runtime.
401  //
402  // If m_transparency_id is not nil, it is the id of another
403  // texture in the ON_Material.m_textures[] array that has
404  // type m_transparency_texture. The runtime bitmap_texture's
405  // alpha is set to (255-max(R,G,B)) (the "value" in the hue,
406  // saturation,value sense) of the correspondeing
407  // transparency_texture pixel.
408  //
409  // For example, if you had a bitmap texuture that was green
410  // with purple dots saved in a RGB .bmp file and you wanted
411  // the purple dots to be semi-transparent, you could create
412  // another bitmap that was black, where the original was green,
413  // and gray, where the original was purple, have an
414  // transparency_texture reference the white/gray bitmap,
415  // and have the bitmap_texture's m_transparency_id
416  // reference the transparency map.

426  // If the m_mode is blend_texture, then m_blend_A[]
427  // and m_blend_RGB[] determine the blending function.
428  //
429  // new alpha = m_blend_constant_A
430  // + m_blend_A[0]*(current alpha)

432  // + m_blend_A[2]*min(current alpha,texture alpha)

437  // + m_blend_RGB[2]*min(current RGB,texture RGB)
438  // + m_blend_RGB[3]*max(current RGB,texture RGB)
439  //
