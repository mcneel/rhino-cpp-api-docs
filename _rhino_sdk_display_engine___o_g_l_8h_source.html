381 // Texture caching and updating...
382 //
383 // Note: The use of these functions must be within the scope of a frame
384 // buffer update. For example: Inside any of the pipeline draw
385 // channels. Using them outside of this scope can/will have
386 // unpredictable results and behavior.
387 
388 // Description:
389 // Returns the handle to a cached texture the engine will use for the given name.
390 //
391 // Parameters:
392 // Name - [in] The name of the cached texture. This can be a name in any format,
393 // consisting of any alpha-numeric characters. The idea is that you choose
394 // names that can easily be used for unique identification.
395 //
396 // Note: Do not use file names as your cached texture names, otherwise you risk the
397 // chance of colliding with textures that have already been or may be loaded
398 // by other materials used in the scene. There are other factors used by Rhino
399 // when caching textures, so the chances are low for collisions...but there
400 // still exists a possibility that textures cached by the use of this method
401 // can collide with textures cached by Rhino's material management that contain
402 // the same file name(s)...so it's probably best to avoid it by not using file
403 // names as your caching name.
404 //
405 // Note2: Holding onto the cached value is not recommended since Rhino can decide to
406 // free up any OpenGL resources it needs to for any reason at any time.
407 // For example: Loading a new file will cause Rhino to destroy all cached OpenGL
408 // resources, including all of your cached textures. Which means any handles
409 // previously returned by this method will no longer work, or may collide with
410 // handles created during the loading of the new file. Therefore it is best to
411 // just always call RhinoGetCachedTexture() before using any of the other texture
412 // caching methods.
413 //
414 // Returns:
415 // 0 - if the engine was unable to locate or create a texture with the given name.
416 // > 0 - Value of cached texture image. Note: The only thing you can do with this value
417 // is use it as a parameter to the other texture caching methods. Do not assume
418 // it can be used for anything else.
419 //
420 RHINO_SDK_FUNCTION
421 unsigned int RhinoGetCachedTexture(const ON_wString& Name);
422 
423 // Description:
424 // Updates the given cached texture's contents with the contents of the passed in Dib.
425 // Note: If the resolution of the Dib is not the same as the cached texture's, then
426 // a new cached texture will be created in its place, re-using the same cached
427 // handle. For performance purposes, it is recommended that you try to manage
428 // using matching resolutions when updating cached textures.
429 //
430 // Parameters:
431 // hTexture - handle to a valid cached texture (obtained by calling GetCachedTexture())
432 // Dib - Image contents used to update the cached texture's contents.
433 //
434 // Returns:
435 // True - if/when the updating of the cached texture succeeds.
436 // False - if/when something failed during the update process.
437 RHINO_SDK_FUNCTION
438 bool RhinoUpdateCachedTextureFromDib(unsigned int hTexture, const CRhinoDib& Dib);
439 
440 // Description:
441 // Updates the given cached texture's contents with the contents of the passed in file
442 // name. Obviously the file name must represent one of the many image file formats
443 // currently supported by Rhino.
444 //
445 // Note: Calling this method can cause performance hits since it reads in files from disk.
446 // Therefore it is important that you keep track of when a cached texture needs
447 // updating, and only update it when necessary and not just blindly call the update
448 // method for every frame update.
449 //
450 // Parameters:
451 // hTexture - handle to a valid cached texture (obtained by calling GetCachedTexture())
452 // Filename - Name of an existing and supported image file on disk.
453 //
454 // Returns:
455 // True - if/when the updating of the cached texture succeeds.
456 // False - if/when something failed during the updating process or if the file could
457 // not be opened or is an invalid/unsupported image file format.
458 RHINO_SDK_FUNCTION
459 bool RhinoUpdateCachedTextureFromFile(unsigned int hTexture, const ON_wString& Filename);
