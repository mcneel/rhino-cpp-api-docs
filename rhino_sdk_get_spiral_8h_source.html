201 /*
202 Description:
203  Low-level function to calculate the C1 cubic NURBS approximation
204  of a helix or spiral.
205 Parameters:
206  axis_start - [in] Helix's axis starting point or center of spiral.
207  axis_dir - [in] Helix's axis vector or normal to spiral's plane.
208  radius_point - [in] Point used only to get a vector that is perpedicular to
209  the axis. In particular, this vector must not
210  be (anti)parallel to the axis vector.
211  pitch - [in] = 0 : Spiral
212  > 0 : Distance between helix's "threads"
213  turn_count - [in] != 0: Number of turns in spiral or helix.
214  > 0 : Right handed or counter-clockwise orientation.
215  < 0 : Left handed or clockwise orientation.
216  Note, for a helix, turn_count * pitch = length of the helix's axis.
217  radius0 - [in] Starting radius.
218  radius1 - [in] Ending radius.
219  curve_out - [out] The output NURBS curve.
220 Returns:
221  True if successful, false otherwise.
222 Remarks:
223  For a helix, you may have r0 == r1.
224  For a spiral r0 == r1 produces a circle.
225  Zero and negative radii are permissible.
226 */

239 /*
240 Description:
241  Low-level function to calculate the C2 non-rational uniform cubic NURBS
242  approximation of a swept helix or spiral
243 Parameters:
244  rail - [in] The rail curve.
245  rail_t0 - [in] Starting portion of rail curve's domain to sweep along.
246  rail_t1 - [in] Ending portion of rail curve's domain to sweep along.
247  radius_point - [in] Point used only to get a vector that is perpedicular to
248  rail'(rail_t0). So (start_point-rail(rail_t0)
249  must not be parallel to rail'(rail_t0).
250  pitch - [in] The pitch (see Remarks).
251  turn_count - [in] The turn count (see Remarks).
252  radius0 - [in] Starting radius (see Remarks).
253  radius1 - [in] Ending radius (see Remarks).
254  points_per_turn - [in] Number of points to intepolate per turn. Vlaue must be
255  >= 4. 12 seems to work well.
256  curve_out - [out] The output NURBS curve
257 Returns:
258  True if successful, false otherwise.
259 Remarks:
260  If turn_count != 0, then the resulting helix will have this many turns.
261  If turn_count == 0, then pitch must be != 0 and the approximate distance
262  bewteen turn will be set to pitch. Use positive turn_count/pitch for
263  right handed helices and negative turn_count/pitch for left handed helices.
264  At least radii one must be nonzero. Negative values are allowed.
265 */
