18 /*
19 Description:
20  Find the extension of the filename passed in as sPath.
21  This function is required to properly handle the name.ext.bak
22  files Rhino 4 and early beta versions of Rhino 5 created.
23 Parameters:
24  sPath - [in]
25  A null terminated string that contains the path to search.
26 Returns
27  * If the sPath parameter is null, then a null is returned.
28  * If no extension is found, then a null is returned (this
29  is different than Windows' PathFindExtension() which
30  returns a pointer to the null terminator in sPath.
31  * If the sPath parameter is not a null and a valid extension
32  exists, then the address of the "." that precedes the extension
33  is returned.
34 Remarks:
35  The RhinoPathFindFileExtension() function works like the
36  as Windows' PathFindExtension(), with three exceptions:
37  1) RhinoPathFindFileExtension() will return ".ext.bak" when
38  sPath is "filename.ext.back" while Windows' PathFindExtension()
39  will return ".bak".
40  2) RhinoPathFindFileExtension() returns null in all other
41  cases.
42  3) RhinoPathFindFileExtension() dow not limit path names
43  to MAX_PATH characters.
44 
45  Note: Valid Windows extensions cannot contain spaces.
46 */
47 RHINO_SDK_FUNCTION
48 const wchar_t* RhinoPathFindFileExtension( const wchar_t* sPath );
